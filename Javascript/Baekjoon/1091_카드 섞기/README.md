# [1091_카드 섞기](https://www.acmicpc.net/problem/1091)

# 처음 생각한 풀이법

- 실제로 카드를 섞어가며 카운트
    - 카드의 순서가 맨 처음 상태로 돌아오면 원하는 대로 섞을 수 없는 것이므로 -1 출력
    - cards의 원소는 카드의 번호이고, 카드가 cards의 인덱스 순서로 쌓여있음을 의미

# 틀렸다면 이후 풀이 방법 및 참고 자료

- 처음에는 문제를 잘못 이해했다. P배열과 cards 배열이 같아지면 되는 거라고 생각했는데 그게 아니어서 따로 check 함수를 만들었다.
- 아무리 섞어도 카드를 원하는 순서대로 쌓을 수 없는 경우를 알아내는 방법은 처음 생각한 방법이 맞았다. 하지만 배열을 비교하는 방법이 시간이 오래 걸려서 자꾸 시간 초과가 났다.
    - 배열을 그냥 `===`으로 비교하면 항상 false가 나오기 때문에 `.join`을 사용해서 풀려고 했는데 이 방법이 오래 걸렸다.(이렇게 푸는 게 오래 걸려서 시간초과가 나는지 모르고 원래 순서로 돌아오면 원하는 대로 섞을 수 없다고 생각한 것이 틀린 줄 알고 다른 방법을 찾아보려 했다.)
    - `.every`를 이용해 푸니 풀렸다.(어차피 처음 상태가 원소와 인덱스가 같은 것이므로 따로 처음 상태에 대한 배열을 만들고 둘을 비교할 필요가 없었다.)
    - [처음 생각한 방법이 맞는지에 대한 증명](https://injae-kim.github.io/problem_solving/2020/02/20/baekjoon-1091.html)
